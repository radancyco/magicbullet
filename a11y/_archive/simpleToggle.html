The following hooks may be used to generate accessible toggle buttons on any web page that leverages the magic bullet script.

CSS

Styles marked "required" are necessary for the interface to function. Feel free to edit these styles in your browser.

HTML

To prepare your button, just add the following data attributes to your button element.

<button data-a11y-button data-a11y-target="my-content">Click Me!</button>

The data-a11y-button attribute initiates the function, while the data-a11y-target attribute tells the functionality which element to toggle. Both of these elements are required. The data-a11y-target attribute value, like the id attribute, must always be unique.

To set up your toggle area, add an id attribute. The value must match the value used in the data-a11y-target attribute of your button. Also, add a data-a11y-content attribute. This is used as a hook by the script.

<div id="my-content" data-a11y-content>

  ...

</div>

Result

A fully accessible button that can toggle content.

Headings

These attributes can be applied to headings, as well. For example, lets say we have a scenario like this:

<h2>Locations</h2>

<div>

  ...

</div>

If we would like to enhance this content and allow it to toggle, then we just add the same attributes as above, like so: 

<h2 data-a11y-button data-a11y-target="about-locations">Locations</h2>

<div id="about-locations" data-a11y-content>

  ...

</div>

Which will render the following:

<h2><button aria-expanded="false" aria-controls="about-locations" class="a11y-button">Locations</button></h2>

<div id="about-locations" class="a11y-content">

  ...

</div>

Result

A fully accessible heading button that can toggle content.

Faux Buttons

If one needs to replace a hyperlink, then the right thing to do is to swap out that link with a button. However, if you are short on time, the same attributes above can be used on hyperlinks as well.

<a href="#" class="btn-cupcake" data-a11y-button data-a11y-target="cupcakes">Cupcake Recipe</a>

<div class="content-cupcake" id="cupcakes" data-a11y-content>

  ...

</div>

Which will remove the hyperlink altogether and replace with a button:

<button aria-expanded="false" aria-controls="cupcakes" class="btn-cupcake a11y-button">Cupcake Recipe</button>

<div id="cupcakes" class="content-cupcake a11y-content">

  ...

</div>

Accessibility

Hyperlinks are often used to masquerade as buttons. This is bad for accessibility. When links are used as buttons, assistive technology may read them out as links, which can be confusing. Links are supposed to take us to other destinations, while buttons should perform actions. Disabled users also expect buttons to work with the enter key and space bar. Click events applied to hyperlinks only work with the enter key, while click events applied to buttons work with both the enter key and the space bar. That's cool. Browsers are smart.

Result

A now fully accessible button that can toggle content.
