This patch addresses various accessibility issues across all Career Sites. This is a temporary solution until these issues can be resolved by the product team. If there are any questions or concerns with the script, please reach out to {{ site.owner-name }}

Implementation

{% highlight html %}{% endhighlight %}

On career sites without it, add MagicBullet under Settings > General > Footer, not in Themes. On Legacy sites, go to Settings > General > Pixel Tracking. This reduces script conflicts and simplifies maintenance. On BaseGST sites, MagicBullet is added by default, so no action is needed. After adding the script, check the console for the following:

If the script is running, smoke-test your pages to ensure the site's core functionality still works. MagicBullet only addresses issues under the hood that relate to accessibility, but minor rendering issues can occasionally occur.

If these messages are missing, the Accessibility Patch isn't loading and you'll need to troubleshoot. If you've installed A11y Pulse, running it will also indicate whether the MagicBullet Accessibility Patch is active.

Troubleshooting

  1. Ensure only one instance of MagicBullet loads per page. Multiple scripts can prevent associated features, like the Accessibility Patch, from running.
  2. Older MagicBullet versions suggested loading via the DOM. If a conflict occurs, check whether the script is created dynamically. If it exists in your JavaScript, you can likely remove it.
  3. Razor commenting isn't supported in Settings. If an older version of MagicBullet exists there, use HTML comments to remove it.
