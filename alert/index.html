Prerequisites

Magic Bullet - https://tmpworldwide.dev/tmp-magic-bullet/

Implementation

COVID-19

We begin with a clean installation of the Magic Bullet script. However, before implementation, please ensure that the Magic Bullet script is not already running on a site. If it is already running with GDPR, CCPA, or A11y, then all you need to do is include the following data attributes to get started:

  • data-alert="true" - This will initiate the alert script.
  • data-alert-covid (boolean, no value required) - This will load the built in, COVID-19 message.

Example:

<script id="tmp-magic-bullet" data-alert="true" data-alert-covid src="https://service.tmpwebeng.com/magicbullet"></script>

Result: https://www.screencast.com/t/sZaHyNxno

Custom Message

If you wish to include a custom message, then replace data-alert-covid with:

  • data-alert-message - Accepts any valid html needed to produce a custom message.

Example:

<script id="tmp-magic-bullet" data-alert="true" data-alert-message=”<h2>Hello World</h2> <p>Now is the time for all good TMP’ers to get down and boogie.</p>" src="https://service.tmpwebeng.com/magicbullet"></script>

Result: https://www.screencast.com/t/OiMsH67Zt

Custom Styles

If you wish to style the modal to suit a client’s brand guidelines, then you can turn off many of the default CSS properties and roll your own:

  • data-alert-remove-css (boolean, no value required) - Removes many default CSS properties from alert (modal and banner). Once removed, theme level CSS will take over, which you can edit as you see fit.

Example:

<script id="tmp-magic-bullet" data-alert="true" data-alert-message="..." data-alert-remove-css src="https://service.tmpwebeng.com/magicbullet"></script>

Result: https://www.screencast.com/t/5D9qcgzbUt
