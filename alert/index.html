Prerequisites

Magic Bullet - https://tmpworldwide.dev/tmp-magic-bullet/

Implementation

COVID-19

We begin with a clean installation of the Magic Bullet script. However, before implementation, please ensure that the Magic Bullet script is not already running on a site. If it is already running with GDPR, CCPA, or A11y, then all you need to do is include the following data attributes to get started:

  • data-alert="true" - This will initiate the alert script.
  • data-alert-covid (Boolean, no value required) - This will load the built in, COVID-19 message.

Example:

<script id="tmp-magic-bullet" data-alert="true" data-alert-covid src="https://services.tmpwebeng.com/magicbullet/"></script>

Result: https://www.screencast.com/t/sZaHyNxno

Custom Message

If you wish to include a custom message, then replace data-alert-covid with:

  • data-alert-message - Accepts any valid html needed to produce a custom message.

Example:

<script id="tmp-magic-bullet" data-alert="true" data-alert-message="<h2>Hello World</h2> <p>Now is the time for all good TMP'ers to get down and boogie.</p>" src="https://services.tmpwebeng.com/magicbullet/"></script>

Result: https://www.screencast.com/t/OiMsH67Zt

Custom Styles

If you wish to style the modal to suit a clientâ€™s brand guidelines, then you can turn off many of the default CSS properties and roll your own:

  • data-alert-remove-css (Boolean, no value required) - Removes many default CSS properties from alert (modal and banner). Once removed, theme level CSS will take over, which you can edit as you see fit.

Example:

<script id="tmp-magic-bullet" data-alert="true" data-alert-message="..." data-alert-remove-css src="https://services.tmpwebeng.com/magicbullet/"></script>

Result: https://www.screencast.com/t/5D9qcgzbUt

Banner Mode

If you wish to load the alert as a banner to be appended to the very top of the page, use:

  • data-alert-banner (Boolean, no value required) - Appends a banner to the top of the page instead of showing a modal window. The same data attributes that are used to control the modal window, can be used here as well.

Example:

<script id="tmp-magic-bullet" data-alert="true" data-alert-banner src="https://services.tmpwebeng.com/magicbullet/"></script>

Result: https://www.screencast.com/t/reISBtAK

Banner Target (New)

If you wish to customize placement of the banner, you may do so by creating an empty div element with a unique id value. This element can then be placed anywhere on the page you wish for message to appear. This id can then be refrenced via the following parameter:

  • data-alert-banner-target="[DIV ID VALUE]" - Accepts value of id on element you are targeting.

Example:

<div id="my-alert"></div>

...

<script id="tmp-magic-bullet" data-alert="true" data-alert-banner data-alert-banner-target="my-alert" src="https://services.tmpwebeng.com/magicbullet/"></script>

Result: https://www.screencast.com/t/tpNLwLutB

Note: May only be used with data-alert-banner. Also, if you wish for message to remain persistent, then data-alert-bypass-cookie must also be used.

Persistent Banner

By default, the banner will always show a close button and can be dismissed in the same way that the modal can. If you wish to have the banner persist, then use:

  • data-alert-bypass-cookie (Boolean, no value required) - Bypasses default behavior of including a button to close banner and set cookie.

Example:

<script id="tmp-magic-bullet" data-alert="true" data-alert-banner data-alert-bypass-cookie src="https://services.tmpwebeng.com/magicbullet/"></script>

Result: https://www.screencast.com/t/7pAsb9CUU

Close Button Text

  • data-alert-button-text (string) - Overrides the default close button value, which is "Close".

Example:

<script id="tmp-magic-bullet" data-alert="true" data-alert-covid data-alert-remove-css data-alert-button-text="OK" src="https://services.tmpwebeng.com/magicbullet/"></script>

Result: https://www.screencast.com/t/11oN8VJI
