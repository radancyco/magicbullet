This script may be used to generate a California Consumer Privacy Act (CCPA) notice on any Radancy hosted website. Due to privacy concerns, California residents who engage with any website that collects data, via cookies, must now be warned about this practice in advance and be able to opt-in, if necessary, to permit this data to be used. You can learn more about the CCPA on the official website.

Note: This document is for internal Radancy use only.

Implementation

To adhere to this regulation, a notice must be included on every page of your site. The following script may be implemented on a site to achieve this goal. Which way you choose to implement the script, largely depends on the size of your site and/or how it has been templated. In most cases, implementation should not take more than 30-60 minutes per site.

HTML

If you require HTML, add the following script before the closing body tag of your page or template:

<script id="tmp-magic-bullet" data-ccpa="true" data-ccpa-client-name="[client name]" data-ccpa-policy-url="[site policy url]" src="https://services.tmpwebeng.com/magicbullet/"></script>

JavaScript

If your site contains many templates, but leverages the same, single JavaScript file, implementing a sitewide notice may be more easily achieved by adding the following DOM (Document Object Model) method to your global script: 

var magicBullet = document.createElement("script");
magicBullet.setAttribute("id", "tmp-magic-bullet");
magicBullet.setAttribute("data-ccpa", "true");
magicBullet.setAttribute("data-ccpa-client-name", "[client name]");
magicBullet.setAttribute("data-ccpa-policy-url", "[site policy url]");
magicBullet.setAttribute("src", "https://services.tmpwebeng.com/magicbullet/");
document.body.appendChild(magicBullet);

Note: The data-ccpa, data-ccpa-client-name and data-ccpa-policy-url attributes are mandatory parameters.

Banner Parameters

The following is a list of parameters that can be passed to the script to exhibit more control over the notice.

Mandatory

The following parameter(s) are mandatory.

Script Initiation

Passing this data attribute a value of "true", will initiate the CCPA script.

HTML
data-ccpa="true"
JavaScript
magicBullet.setAttribute("data-ccpa", "true");

Client Name

Passing this data attribute a string value, such as the name of the client, will include that value in any needed legal verbiage, such as the notice placed within TalentBrew's Data Collection Forms.

HTML
data-ccpa-client-name="Clarity Robotics"
JavaScript
magicBullet.setAttribute("data-ccpa-client-name", "Clarity Robotis");

Policy URL

Passing this data attribute a policy URL, will include that URL within the notice and inline form message. The value must be in the form of a fully qualified URL. Protocol-relative URL's should not be used. If the privacy policy is local to the domain, then please use a relative path instead—e.g., "/privacy/" instead of "https://www.clarityrobotics.com/privacy/".

HTML
data-ccpa-policy-url="https://www.clarityrobotics.com/privacy/"
JavaScript
magicBullet.setAttribute("data-ccpa-policy-url", "https://www.clarityrobotics.com/privacy/");

Translation

The following parameter can be used to change the language of the CCPA notice and the TalentBrew Data Collection Form message.

Language

Passing this data attribute a language code, will translate the notice and form message into the desired language. The default language is English.

HTML
data-ccpa-language="es"
JavaScript
magicBullet.setAttribute("data-ccpa-language", "es");
Language Codes

The following language code values are accepted by the script:

  • es (Spanish)

Note: If new languages are required, please contact Dan Green.

Visual Formatting

Warning: Please do not attempt to override the banner notice with CSS or JavaScript at the implementation level. Local overrides may interfere with future formatting changes required within the CCPA script itself. If you run into any formatting issues that can't be solved with any available parameters the CCPA script offers, then please contact Michael Spellacy for assistance. The inline form message may be safely manipulated with CSS.

The following parameter(s) can be used to apply basic formatting to the notice. All parameters are optional:

Banner Position

Passing this data attribute a value of "true", will will position the CCPA banner at the top of the page. The default position is to load at the bottom of the page.

HTML
data-ccpa-banner-top="true"
JavaScript
magicBullet.setAttribute("data-ccpa-banner-top", "true");

Button Color

Passing this data attribute any valid CSS color value will change the background color of the "Accept" or "Close" button.

HTML

JavaScript

Button Text (Accept)

Passing this data attribute any string value will change the text of the "Accept" button. The default text is "Accept".

HTML

JavaScript
magicBullet.setAttribute("data-ccpa-accept-button-text", "Submit");

Button Text (Close)

Passing this data attribute any string value will change the text of the "Close" button. The default text is "Close".

HTML

Button Text Color

Behavior

The following parameter(s) can be used to apply or remove basic behaviors to the notice. All parameters are optional:

Banner Expiration

Passing this data attribute a value of "true", will allow the banner to expire after one day, instead of the default 30 days.

Note: This attribute should only be used with Explicit Opt-In

HTML
data-ccpa-date-expire="true"
JavaScript
magicBullet.setAttribute("data-ccpa-date-expire", "true");

Banner Removal (New)

Passing this data attribute a value of "true", will prevent the banner notice from loading. Implied consent will also not be captured. Dynamically generated form messages will still be applied to all forms on the site.

HTML
data-ccpa-banner-remove="true"
JavaScript
magicBullet.setAttribute("data-ccpa-banner-remove", "true");

Custom Message

Passing this data attribute a string value will override the default message. HTML may be included.

HTML
data-ccpa-custom-message="<h2>About Cookies</h2> <p>We use cookies to help make this website better, to improve our services and for advertising purposes. You can learn more about our use of cookies by <a href='https://privacy.thewaltdisneycompany.com/en/current-privacy-policy/cookies-policy/'>clicking here</a>.</p>"
JavaScript
magicBullet.setAttribute("data-ccpa-custom-message", "<h2>About Cookies</h2> <p>We use cookies to help make this website better, to improve our services and for advertising purposes. You can learn more about our use of cookies by <a href='https://privacy.thewaltdisneycompany.com/en/current-privacy-policy/cookies-policy/'>clicking here</a>.</p>");

Cross-Domain Sharing

There are times when a client may wish to share cookie values and ensure the banner is hidden across multiple URL's with the same domain or subdomain. This is possible if the Magic Bullet script is placed on all needed sites and each include the following data attribute:

HTML
data-ccpa-domain=".domain.com"
JavaScript
magicBullet.setAttribute("data-ccpa-domain", ".domain.com");

Explicit Consent

If your site requires that consent to track and use cookies be explicitly given, then you may use this data attribute to accomplish that. Passing this data attribute a string value of "true" will override the default behavior, which is set to passive consent, an implied acceptance via the continued use of the website. The banner will now persist from page to page until the user explicitly clicks on the "Accept" button in the banner.

Note: This attribute should only be used with Explicit Opt-In

Note: If explicit consent is used, it is very likely that the SEO team will need to be adjusting desired tracking behaviors in this scenario, so be sure to remind your DPM of this task.

HTML
data-ccpa-explicit-consent="true"
JavaScript
magicBullet.setAttribute("data-ccpa-explicit-consent", "true");

Explicit Opt-in (New)

As it stands currently, the CCPA banner's primary purpose is to alert users to a client's privacy practices. However, if a client collects and shares information with third-parties, then user's will be required to explicitly opt-in. To enable this behavior, just add the following:

HTML
data-ccpa-opt-in="true"
JavaScript
magicBullet.setAttribute("data-ccpa-opt-in", "true");

New Window

Passing this data attribute a value of "false" will disable the privacy policy link from opening in a new window. The default behavior is for all notice links to open in a new window. These visual cues will not appear when new windows are disabled.

New Window Icons

The visual cue ( ) associated with these links are a usability and accessibility best practice that serve to indicate that the links will spawn a new tab or window. They cannot be changed or removed at this point in time, unless new windows are disabled. You can read more about this best practice in Radancy's code standard guidelines.

HTML
data-ccpa-new-window="false"
JavaScript
magicBullet.setAttribute("data-ccpa-new-window", "false");

TalentBrew Data Collection Forms

The second task of this script is to include a privacy policy statement on all TalentBrew forms that collect data. By default, the script will look for the .data-form form-field.submit class and prepend the following element, and message, before the form's "Submit" button:

<p class="form-field ccpa-eu-tmp-notice">By submitting your information, you acknowledge that you have read our privacy policy (this content opens in new window) and consent to receive email communication from [company name].</p>

Parameters

The following parameter(s) are optional:

Custom Form Message

Passing this data attribute a string value will override the default form message. HTML may be included.

HTML
data-ccpa-custom-form-message="By submitting your information, you acknowledge that you have read our <a href='/privacy/' target='_blank'>candidate privacy policy</a> and consent to receive email communication."
JavaScript
magicBullet.setAttribute("data-ccpa-custom-form-message", "By submitting your information, you acknowledge that you have read our <a href='/privacy/' target='_blank'>candidate privacy policy</a> and consent to receive email communication.");

Form Message Exception

There may be instances where one may only require the CCPA message to load on certain forms and not others. By default, the script looks for the .data-form class and applies the message, but if you wish to exclude the message, simply add data-ccpa-form-msg-exception to the opening form element.

HTML
<form class="data-form" ... data-ccpa-form-msg-exception> ... </form>

Form Message Position

If you need to override the default behavior of placing the form message over the submit button, then you may include the following data attribute and value.

HTML
data-ccpa-form-bypass="true"
JavaScript
magicBullet.setAttribute("data-ccpa-form-bypass", "true");

Removing the default behavior means that you will need to manually insert the message into the desired position within the form. To do this, place the following markup wherever you would like the message to appear. The surrounding p tag can be replaced with a span or any other tag of your choosing, but please be sure to keep the ccpa-eu-tmp-notice class name in place, as the message will still be inserted into the tag, via JavaScript. The "form-feild" class may be removed, if needed.

Note: CSS formatting may be safely applied to this tag, as needed.

<p class="form-field ccpa-eu-tmp-notice"></p>

Styling Hook

Please be aware that a CSS hook is being added to the body tag of each page the script loads on, which could come in handy for any CCPA related formatting issues/needs that may come up. The name of the class is magic-bullet-ccpa.
