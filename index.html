A script to chase all your troubles away.

Magic Bullet is a control, or placeholder script with one single purpose in mind — to load other scripts. On it's own, the script will do nothing until you start passing parameters into it. This allows you to load different scripts in for different client needs. For example, let's say we want to load in a script that can fix accessibility issues on a client's site. To do that, all you would need to do is add the following paramater to the Magic Bullet script:

<script id="tmp-magic-bullet" data-a11y="true" src="https://services.tmpwebeng.com/magicbullet/"></script>

This will append and initiate the following script:

<script id="a11y-fixes" src="https://services.tmpwebeng.com/magicbullet/a11y/prod.js"></script>

Why?

Managing scripts in this manner allows us to maintain multiple enhancements from a single location. The more sites we include Magic Bullet on, the more leverage the implementation team will have in addressing unforseen future needs or issues that occur across multiple sites. Rather than asking the implementation team to update multiple sites, we could, potentially, apply a single solution, which will affect all of the sites that Magic Bullet is included on.
